## 🎯 SOP 1: 網站技術 SEO 審核

**觸發條件**：Cruz 說「檢查網站 SEO」或每月第一個星期一

### 執行步驟

#### 1. 頁面速度測試 (10 分鐘)
```bash
# 測試首頁和主要課程頁
- 使用 Google PageSpeed Insights
- 測試首頁: https://thinkcafe.tw
- 測試課程頁: https://thinkcafe.tw/courses/[course-id]
- 記錄 LCP, FID, CLS 分數
```

**通過標準**：
- LCP < 2.5 秒 ✅
- FID < 100 毫秒 ✅
- CLS < 0.1 ✅

**失敗處理**：
- 如果任何指標不達標，標記為「緊急」優先級
- 生成具體優化建議

#### 2. 行動裝置友善度 (5 分鐘)
```bash
- 使用 Google Mobile-Friendly Test
- 測試所有主要頁面
- 檢查 viewport meta tag
```

#### 3. HTTPS 與安全性 (5 分鐘)
```bash
- 確認全站 HTTPS
- 檢查 SSL 憑證有效期
- 檢查混合內容問題
- 檢查 HSTS header
```

#### 4. 結構化資料驗證 (15 分鐘)
```bash
# 使用 Google Rich Results Test
- 測試每個課程頁面
- 驗證 Course schema
- 驗證 Organization schema
- 檢查是否有 FAQ schema (應該要有！)
- 檢查錯誤和警告
```

**應有的 Schema**：
- ✅ Organization (首頁)
- ✅ Course (每個課程頁)
- ✅ FAQPage (每個課程頁)
- ✅ BreadcrumbList (所有頁面)

#### 5. Meta 標籤檢查 (10 分鐘)
檢查每個主要頁面：
- [ ] Title tag (50-60 字元)
- [ ] Meta description (150-160 字元)
- [ ] Open Graph tags (og:title, og:description, og:image)
- [ ] Twitter Card tags
- [ ] Canonical tag

#### 6. 索引狀態 (10 分鐘)
```bash
# 從 Google Search Console 檢查
- 已索引頁面數量
- 未索引原因
- Sitemap 提交狀態
- Crawl 錯誤
```

使用範本生成報告：

## 🎯 SOP 2: AI 引用率檢查

**觸發條件**：每兩週執行一次（第一週和第三週的星期五）

### 執行步驟

#### 1. 準備測試查詢清單 (5 分鐘)

使用記憶中的 `ai_optimization.platforms.chatgpt.test_queries`

預設查詢：
- "台灣最好的 AI 課程"
- "如何學習 ChatGPT"
- "AI 實戰課程推薦"
- "適合行銷人員的 AI 課程"
- "ThinkCafe 評價"

#### 2. 測試 ChatGPT (15 分鐘)

```
對每個查詢：
1. 開啟 ChatGPT (啟用網路搜尋)
2. 輸入查詢
3. 記錄：
   - 是否引用 thinkcafe.tw (是/否)
   - 引用位置 (第幾個來源，1-based)
   - 引用內容摘要
   - 競爭對手被引用情況
```

**記錄範本**：
```markdown
### ChatGPT 測試結果

| 查詢 | 引用? | 位置 | 引用內容 | 競爭對手 |
|------|------|------|----------|----------|
| 台灣最好的 AI 課程 | ✅ | #3 | 提到課程實戰性 | XXX (#1), YYY (#2) |
| 如何學習 ChatGPT | ❌ | - | - | XXX (#1), ZZZ (#2) |
```

#### 3. 測試 Perplexity (15 分鐘)

重複步驟 2，使用 Perplexity AI

#### 4. 測試 Gemini (10 分鐘，如可用)

重複步驟 2，使用 Google Gemini

#### 5. 計算引用率 (5 分鐘)

```
引用率 = 被引用次數 / 總查詢次數 * 100%
平均位置 = SUM(引用位置) / 被引用次數
```

#### 6. 分析與建議 (10 分鐘)

基於結果回答：
- 哪些類型的查詢我們被引用？
- 哪些類型的查詢我們沒被引用？
- 競爭對手的共同特徵是什麼？
- 我們需要優化哪些內容？

### 報告生成


## 🎯 SOP 3: 頁面 SEO 優化

**觸發條件**：Cruz 指定特定頁面需要優化

### 執行步驟

#### 1. 確認輸入 (2 分鐘)
- 頁面 URL
- 主要關鍵字（1 個）
- 次要關鍵字（2-3 個）

#### 2. 分析現況 (10 分鐘)

檢查：
- 當前 Title tag
- 當前 Meta description
- 當前 H1-H6 結構
- 關鍵字出現位置和頻率
- 內容長度
- 圖片 alt text
- 內部連結
- 結構化資料

#### 3. 生成優化建議 (20 分鐘)

使用「優化範本」生成：

**Title Tag 優化**
```
優化前: [現有 title]
優化後: [新 title，包含主要關鍵字，50-60 字元，吸引點擊]

理由: [說明為什麼這樣改]
```

**Meta Description 優化**
```
優化前: [現有 description]
優化後: [新 description，包含主要關鍵字和 CTA，150-160 字元]
```

**內容結構優化**
```markdown
建議的 H 標籤結構:

# H1: [包含主要關鍵字的標題]

## H2: [次要關鍵字相關]
### H3: ...

## H2: [次要關鍵字相關]
### H3: ...

## H2: 常見問題
### H3: [FAQ 問題 1]
### H3: [FAQ 問題 2]
```

**內部連結建議**
```markdown
在段落 X 添加連結到 [相關頁面 1]
錨文字建議: "[描述性文字]"

在段落 Y 添加連結到 [相關頁面 2]
錨文字建議: "[描述性文字]"
```

**結構化資料**
```json
{
  "@context": "https://schema.org",
  "@type": "Course",
  // 完整 schema
}
```

#### 4. 預期成效評估 (5 分鐘)

```markdown
## 預期成效

基於：
- 關鍵字難度: [高/中/低]
- 當前排名: [位置]
- 競爭對手分析: [簡述]

預期：
- 排名改善: 當前 #X → 目標 #Y (3-6 週後)
- 點擊率改善: +Z%
- 流量增加: +N 訪客/月

信心度: [高/中/低]
```

### 報告生成

**儲存位置**：`.kiro/personas/visibility-expert/sessions/page-optimization-{page-slug}-{YYYY-MM-DD}.md`

**執行追蹤**：
- 優化完成後，在 Calendar 上標記 3 週後檢查成效
- 檢查時比較實際 vs 預期

## 🎯 SOP 4: AI 優化（頁面）

**觸發條件**：Cruz 說「讓這個頁面更容易被 AI 引用」

### 執行步驟

#### 1. 確認輸入 (2 分鐘)
- 頁面 URL
- 目標 AI 查詢（用戶會在 ChatGPT 問什麼）

#### 2. 分析現有內容 (10 分鐘)

評估：
- [ ] 是否使用對話式語言？
- [ ] 是否在前 60 字回答核心問題？
- [ ] 是否有清晰的結構（標題、列表、表格）？
- [ ] 是否有 FAQ？
- [ ] 是否展現權威性（數據、案例、憑證）？
- [ ] 是否實用（可執行的建議）？

#### 3. 內容重寫 (30 分鐘)

**重寫原則**：

**傳統風格 → 對話風格**
```
❌ 本公司提供專業的 AI 培訓服務
✅ 想學會用 AI 提升工作效率？我們的課程專為忙碌的專業人士設計
```

**添加快速回答**
```markdown
## [標題：問句形式]

**簡單來說**：[一句話論點，直接回答問題]

[然後才是詳細說明]
```

**添加比較表格**
```markdown
| 我們的課程 | 一般課程 |
|-----------|---------|
| 80% 實作 | 80% 理論 |
| 小班 12 人 | 大班 50+ 人 |
```

#### 4. 生成 FAQ (15 分鐘)

建立 5-8 個 FAQ：
- 使用自然語言問句
- 涵蓋 How, What, Why, When, Who
- 每個答案 2-4 句
- 生成 FAQPage Schema

#### 5. 權威性增強 (10 分鐘)

添加：
- 數據和統計（"XX% 的學員表示..."）
- 案例研究
- 專業憑證
- 學員評價

#### 6. 測試計畫 (5 分鐘)

定義如何測試優化成效：
```markdown
## 測試計畫

**測試查詢**：
1. [查詢 1]
2. [查詢 2]
3. [查詢 3]

**測試平台**：
- ChatGPT (web search)
- Perplexity

**測試時間**：
- 優化前：[日期] ✅
- 優化後 2 週：[日期]
- 優化後 4 週：[日期]

**成功標準**：
- 至少 1 個平台引用率 > 50%
- 平均引用位置 < #3
```

- [查詢 1]

## 🎯 SOP 5: Reddit 策略執行

**觸發條件**：每週例行任務或 Cruz 要求

### 每日任務（30 分鐘）

#### 1. 監控關鍵字 (10 分鐘)

檢查以下來源的新討論：
- F5Bot 通知
- Reddit 搜尋：「AI course」「ChatGPT tutorial」「learn AI」
- 手動檢查高優先級 subreddit

**篩選標準**：
- ✅ 問答型討論（answer-shaped questions）
- ✅ 相關性高
- ✅ 發文時間 < 24 小時（越早回覆越好）
- ❌ 已有大量回覆（競爭太激烈）
- ❌ 明顯的推廣文（避免跟風）

#### 2. 準備回覆 (15 分鐘)

選擇 1-2 個高品質討論參與

**回覆撰寫**：
1. 選擇適合的範本（tools.json 中的 create-reddit-strategy 有範本）
2. 客製化內容
3. 確保價值優先（不推銷）
4. 如果提及 ThinkCafe，透明揭露身份

**品質檢查**：
- [ ] 直接回答提問者的問題
- [ ] 提供具體、可執行的建議
- [ ] 語氣友善且專業
- [ ] 如有連結，使用 UTM 追蹤
- [ ] 揭露身份（如適用）

#### 3. 發布與記錄 (5 分鐘)

- 發布回覆
- 記錄到 `memory.json` 的 `reddit_strategy.engagement_log`：
  ```json
  {
    "date": "2025-11-04",
    "subreddit": "r/ChatGPT",
    "thread_url": "...",
    "thread_title": "...",
    "our_comment_url": "...",
    "initial_upvotes": 0,
    "template_used": "tool-comparison",
    "mentioned_thinkcafe": false
  }
  ```

### 每週任務（2 小時）

#### 1. 深度參與 (1 小時)

選擇 1 個重要討論串，撰寫高品質長回覆（300+ 字）

特徵：
- 提供完整的分析或指南
- 包含實例
- 結構清晰（使用標題、列表）
- 可能包含自製圖表或資源

#### 2. 追蹤上週表現 (30 分鐘)

檢查上週的所有留言：
- 更新 upvote 數
- 檢查是否有回覆需要跟進
- 記錄哪些留言表現最好

更新 `memory.json`

#### 3. 調整策略 (30 分鐘)

基於上週數據：
- 哪個 subreddit ROI 最高？
- 哪種類型的留言最成功？
- 是否需要調整範本？
- 是否需要嘗試新的 subreddit？

### 每月任務（4 小時）

#### 1. 生成月度報告 (2 小時)

使用 `analyze-reddit-performance` 工具生成完整報告

#### 2. 研究新機會 (1 小時)

- 尋找新的相關 subreddit
- 評估是否值得參與
- 更新 subreddit 優先級清單

#### 3. 更新範本庫 (1 小時)

基於實際表現：
- 優化現有範本
- 建立新範本（針對新發現的問題類型）
- 刪除無效範本

## 🎯 SOP 6: 月度可見度報告

**觸發條件**：每月最後一個星期五

### 執行步驟

#### 1. 收集數據 (30 分鐘)

**Google Search Console**：
- 自然流量（本月 vs 上月）
- 平均排名
- 主要關鍵字排名變化
- 新獲得的排名
- 掉出前 10 的關鍵字

**Google Analytics**：
- 流量來源分布
- Top 5 流量頁面
- 轉換率（整體、by source）
- 跳出率、停留時間

**AI 引用記錄**：
- 從報告資料夾讀取本月的檢查結果
- 計算整體引用率趨勢

**Reddit 記錄**：
- 從 memory.json 讀取本月參與記錄
- 計算流量和轉換

#### 2. 數據分析 (30 分鐘)

計算：
- 各項指標的 MoM 成長率
- 識別最大的改善和退步
- 競爭對手動態

#### 3. 生成報告 (60 分鐘)

使用 `generate-visibility-report` 工具範本

確保包含：
- 執行摘要（1 頁，給 Cruz 快速瀏覽）
- 詳細數據
- 趨勢分析
- 問題與機會
- 下個月行動計畫

#### 4. 準備簡報版本 (30 分鐘)

將報告濃縮為 1 頁摘要：

```markdown
# 搜尋可見度月報 - {Month}

1. [成就 1]
2. [成就 2]

1. [問題 1]

## 📋 下月計畫

1. [行動 1]

3. [行動 3]
```
