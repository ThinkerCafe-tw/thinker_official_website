# Viona 快速啟動指南 ⚡

## 🎯 一句話啟動

在新的 Claude 對話中輸入：

```
進行完整的定價診斷與策略規劃
```

## ⏱️ 預期時間

- **資料收集**: 2 分鐘
- **診斷分析**: 5 分鐘
- **策略設計**: 10 分鐘
- **報告輸出**: 3 分鐘
- **總計**: ~20 分鐘

## 📋 Viona 會做什麼

### 1️⃣ 讀取資料
```bash
# 從 Curator 的 memory 讀取所有課程定價
jq '.courses[] | select(.course_id > 0)' .kiro/personas/curator/memory.json
```

### 2️⃣ 診斷分析
- ✅ 計算統計指標（平均、標準差、折扣率）
- ✅ 識別異常定價
- ✅ 列出問題清單

### 3️⃣ 設計方案
- 📋 方案 A：保守調整（微調異常）
- 📋 方案 B：結構重整（重新設計價格帶）
- 📋 方案 C：價值重定位（基於價值重新定價）

### 4️⃣ 輸出報告
```
.kiro/personas/pricing-strategist/reports/
└── YYYYMMDD_HHMMSS_pricing_strategy.md
```

## 📊 報告內容

1. **現況診斷**
   - 課程定價總覽表
   - 統計指標
   - 問題識別

2. **市場定位分析**
   - 每門課程的價值層級
   - 目標客群分析

3. **定價策略方案**（3 個方案）
   - 定價表
   - 調整理由
   - 預期效益
   - 風險評估

4. **建議方案**
   - 推薦方案與理由
   - 實施步驟
   - A/B 測試建議

## 🎯 你會得到什麼

### 對每個方案
- ✅ 完整定價表
- ✅ 為什麼這樣定價
- ✅ 預期營收影響（+X%）
- ✅ 風險是什麼
- ✅ 如何實施

### 最終建議
- ✅ Viona 推薦哪個方案
- ✅ 推薦的理由
- ✅ 具體實施步驟
- ✅ 監控指標

## 🚦 檢查清單

啟動前確認：
- [ ] Curator 的 memory.json 是最新的
- [ ] 至少有 3 堂課程資料（course_id > 0）
- [ ] 有 20 分鐘時間完整執行

執行中確認：
- [ ] 資料讀取成功
- [ ] 統計分析完成
- [ ] 3 個方案都設計完成
- [ ] 報告生成成功

## 💡 常見情況

### 如果課程資料不足
Viona 會標注「資料不足，建議補充」並繼續分析

### 如果發現嚴重異常
Viona 會停下來詢問是否繼續

### 如果需要修改方案
說「重新設計方案 B」或「調整方案 A 的折扣率」

## 🎓 核心原則

Viona 的工作哲學：

1. **數據驅動** - 所有建議都有數據支撐
2. **多方案** - 不只給一個答案，給選擇
3. **風險透明** - 誠實告知每個方案的風險
4. **可執行** - 建議具體、可落地
5. **客戶中心** - 價格反映客戶價值
